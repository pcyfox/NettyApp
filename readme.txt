lib_xxx:(功能型模块，不涉及任何具体业务相关，可充当工具、媒介、及通用功能的实现等)
凡是lib_xxx模块都是一个个相互独立的模块，它们的功能是独立的且完整的，
当多个lib_xxx都需要引用同一个或多个第三方或谷歌支持库等时只需要使用implementation各自gradle中添加即可，
只要保证相同库的版本一致就不会产生冲突及冗余问题，它们在打包时引用的是同一份代码。
它们都不依赖于其它任何lib_xxx而实现，它们只为业务层的各个module_xxx模块提供服务服务
目的：
1、解耦，可以方便拆装、移植到其它项目
2、易于测试，正因为其独立性，可以很方便的将其配置成一个可以独立运行的APP


module_xxx:(业务型模块，与具体业务息息相关，不同的APP有不同的需求，通常会根据不同需求采用组合不同lib__xxx的方式帮助快速开发)
与各个lib_xxx间关系一样，各个module_xxx间不产生任何直接依赖，如果需要交换数据可以各种事件总成机制来实现，
不得已的话可以通过大家都依赖同一个lib_xxx，以它为桥梁相互沟通（如：项目的lib_event就为此目的而生）
目的：
与lib_xxx设计目的基本相似，但是更要的意义在于帮助协作开发，例如项目中有多个业务模块（如：登录、聊天、购物、玩游戏...）
那么这个应用可以多人同时开发，各模块间只需要协商好数据交换即可，并且由于这种独立性每个人的module_xxx都是一个独立的APP。可以独立运行、独立测试、独立验收、甚至发布。
